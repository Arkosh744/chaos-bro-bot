# Chaos Bro Bot v2 — Брейншторм: фичи, оптимизация, UX

> Дата: 2026-03-17
> Состояние: 5200+ строк, 20+ коммитов, 30+ фич, веб-панель, мультиюзер

---

## I. КРИТИЧЕСКИЕ БАГИ И ПРОБЛЕМЫ

### 1. time.Sleep(2s) блокирует обработчик
**Где:** `handlers.go:341` — bargain sleep
**Проблема:** Блокирует горутину telebot на 2 секунды. При нескольких юзерах — backpressure. Нет thinking-анимации во время ожидания.
**Фикс:** Заменить на `go func() { time.Sleep(2s); startThinking(); ... }()` или отправлять bargain с thinking сразу.

### 2. Web-панель без аутентификации
**Где:** `web/handlers.go` — все эндпоинты открыты
**Проблема:** Любой может читать данные юзеров, менять профиль, отключать scheduler.
**Фикс:** Basic auth с токеном из конфига. `Authorization: Bearer <token>` header.

### 3. SetEnabled не управляет горутинами scheduler
**Где:** `scheduler.go:45-50`
**Проблема:** Toggle через веб-панель меняет флаг, но не останавливает/запускает пинги.
**Фикс:** Проверка `s.cfg.Enabled` внутри loop() перед каждым sendPing(). Или канал для hot-reload конфига.

### 4. Capsule delivery привязана к scheduler.Enabled
**Где:** `scheduler.go:76-79`
**Проблема:** Если scheduler выключен — капсулы не доставляются.
**Фикс:** capsuleLoop запускать всегда, независимо от Enabled.

## II. UX-УЛУЧШЕНИЯ

### 5. `/help` — список всех команд и кнопок
**Сейчас:** Нет команды помощи.
**Надо:** Красиво отформатированный список всех возможностей с эмоджи. Вызов: `/help` + кнопка в `/start`.

### 6. Offended reply — пауза перед основным ответом
**Сейчас:** Обида + ответ мгновенно два сообщения подряд.
**Надо:** Отправить обиду, подождать 3-4с с thinking, потом ответить. Как будто бот "передумал обижаться".

### 7. handleRandomize — состояние "жду вопрос"
**Сейчас:** Кнопка просто шлёт инструкцию, следующее сообщение с `?` — рандомайзер.
**Надо:** Установить состояние "waiting_for_question" в counters. Следующее сообщение (любое) обработать как вопрос рандомайзеру. Сбросить состояние. Убрать автоматический `?`-рандомайзер или сделать его опциональным.

### 8. Mirror mode — статус и досрочное отключение
**Сейчас:** Нет фидбэка сколько сообщений осталось, нет `/mirror off`.
**Надо:**
- В каждом mirror-ответе: маленький постфикс `[🪞 7/10]`
- `/mirror off` или `/mirror` повторно — выключает
- `/mirror` когда уже активен — показать статус

### 9. Silence mode — `/silence off` + точное время
**Сейчас:** Нет отмены, грубое округление часов.
**Надо:**
- `/silence off` — досрочная отмена
- Показывать оставшееся время точнее: "Осталось 2ч 15мин"
- Scheduler не пингует в silence mode

### 10. Breathing — ачивка по факту завершения
**Сейчас:** defer в handleBreathing срабатывает сразу, RunBreathing async.
**Надо:** Вызывать checkAndSendAchievements в конце RunBreathing (передать userID и callback).

### 11. Кнопочная клавиатура — слишком много кнопок
**Сейчас:** 4 ряда по 3 кнопки = 10 кнопок. Перегружено.
**Надо:** 2 уровня:
- Основное меню: `[Очнись] [Хаос] [Судьба]` + `[Ещё ➡️]`
- Доп. меню: `[Зажарь] [Мудрость] [Гороскоп]` + `[Кости] [Зеркало] [⬅️ Назад]`
- Или inline-кнопки для редких команд вместо reply keyboard.

### 12. Onboarding — первое сообщение
**Сейчас:** `/start` — "Йо. Я [имя]. Жми кнопки."
**Надо:** Мини-тур: после `/start` отправить 2-3 сообщения с задержкой:
1. Имя + клавиатура
2. "Я дерзкий друг-трикстер. Не коуч, не AI. Жми кнопки или просто пиши."
3. Первый grounding как welcome-подарок

## III. НОВЫЕ ФИЧИ

### 13. Стикеры и GIF-реакции
**Что:** Бот иногда отвечает стикером или GIF вместо текста (для коротких эмоциональных ответов).
**Как:** Пул из 20-30 стикеров/GIF для разных эмоций. 10% шанс заменить текстовый ответ на стикер.

### 14. Мини-игры в чате
**Что:** Быстрые текстовые игры:
- "Угадай число 1-10" — 3 попытки
- "Правда или действие" — через inline кнопки
- "Ассоциации" — бот начинает слово, юзер продолжает цепочку
**Как:** State machine через counters. Команда `/game` с inline выбором игры.

### 15. Напоминалки в стиле трикстера
**Что:** `/remind 30m выпить воду` — через 30 минут бот напомнит, но в своём стиле.
**Как:** Таблица reminders в SQLite. Горутина проверяет каждую минуту. Claude стилизует напоминание.

### 16. Streak / серия дней
**Что:** Трекинг серии дней подряд когда юзер писал. Бонусы за streak.
**Как:** Счётчик в counters `streak_days` + `last_streak_date`. Ежедневная проверка. Ачивки за 7/30/100 дней.

### 17. Реакции на конкретные эмоджи
**Что:** Если юзер отправляет определённый эмоджи — бот реагирует специально.
**Как:** Расширить EasterEggs на эмоджи-ключи. `❤️` → "Ох, смущён.", `💀` → "RIP. Что случилось?"

### 18. Тематические дни
**Что:** По дням недели бот слегка меняет характер:
- Пн: "Понедельник-хуйнедельник" — максимум сарказма
- Пт: "Пятничный кайф" — более расслабленный
- Вс: "Философское воскресенье" — глубокие вопросы
**Как:** В TimeOfDayMood добавить DayOfWeekMood, инжектить в prompt.

### 19. /compare — сравнение двух юзеров
**Что:** В мультиюзерном режиме — сравнить статистику двух юзеров (кто больше пишет, у кого выше mood, кто больше ноет).
**Как:** Web API `/api/compare?a=ID1&b=ID2`. Бот-команда `/compare @username`.

### 20. Голосовые ответы (TTS)
**Что:** Бот иногда отвечает голосовым (5-10% шанс для коротких ответов).
**Как:** Edge-TTS (бесплатный), генерация .ogg, отправка как Voice. Или Groq TTS если появится.

## IV. ОПТИМИЗАЦИЯ КОДА

### 21. Хелпер claudeReply для дедупликации
**Сейчас:** startThinking + Claude + error fallback повторяется 7+ раз.
**Надо:**
```go
func (b *Bot) claudeReply(c tele.Context, ask func() (string, error), prefix string) error {
    replyFn, stop := b.startThinking(c)
    result, err := ask()
    if err != nil {
        stop()
        return c.Send(prefix+features.RandomFallback(), menu)
    }
    return replyFn(prefix+result, menu)
}
```

### 22. Параллелизация buildUserContext
**Сейчас:** 4 последовательных DB-запроса.
**Надо:** `errgroup.Group` или `sync.WaitGroup` для параллельного выполнения GetSummary + GetLastMessages + GetFactsAsText + GetMessageCount.

### 23. Удалить мёртвый код TricksterReplyWithLevel
**Где:** `trickster.go:18-24`
**Что:** Функция определена, но никогда не вызывается.

### 24. Pre-generate daily lie
**Сейчас:** Ложь генерируется при первом сообщении дня — удваивает латентность.
**Надо:** Генерировать в morningCheckLoop (когда уже вызывается Claude для quest).

### 25. Scheduler: единый buildContext
**Сейчас:** Scheduler дублирует логику buildUserContext.
**Надо:** Вынести buildContext в отдельный пакет или передавать функцию.
